# Design Document: APK Store Frontend

**Author(s):** System Maintainers  
**Status:** Implemented  
**Last Updated:** 2024  

## 1. Overview and Objective

The APK Store is a static, client-side web application designed to act as the user-facing storefront for a modular APK patching framework. It allows users to browse, search, and download community-maintained, patched Android applications. 

The primary objective of the website is to provide a seamless, app-store-like experience without requiring users to navigate the raw GitHub repository or GitHub Releases interface, while maintaining zero backend infrastructure.

## 2. Background

The core repository uses GitHub Actions to automate the downloading, patching, signing, and releasing of Android APKs. While GitHub Releases is an excellent distribution mechanism, it is not user-friendly for non-technical users. A dedicated frontend is required to aggregate release metadata, display application statuses, and provide localized instructions and download links.

## 3. Goals and Non-Goals

### Goals
- **Zero Backend Dependencies:** The site must be 100% static and hosted on GitHub Pages.
- **Rate-Limit Resilience:** Avoid hitting GitHub API rate limits by relying on CI-generated static JSON indexes.
- **Internationalization (i18n):** Native support for English (LTR) and Hebrew (RTL).
- **Responsive & Accessible UI:** Mobile-first design with built-in Dark/Light modes.
- **Frictionless Contributions:** Provide an integrated UI form for requesting new apps that translates into formatted GitHub Issues.

### Non-Goals
- Hosting the actual APK files (delegated to GitHub Releases).
- Server-side rendering (SSR) or complex build steps (e.g., Webpack/Vite).
- User authentication or personalized accounts.

## 4. System Architecture

The frontend is a vanilla JavaScript Single Page Application (SPA) styled with Tailwind CSS via CDN. It operates entirely by consuming static JSON files generated by the repository's CI/CD pipeline.
